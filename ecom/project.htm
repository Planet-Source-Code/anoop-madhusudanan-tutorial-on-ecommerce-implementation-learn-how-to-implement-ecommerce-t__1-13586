1. WHAT WE ARE GOING TO DEVELOP? 
 
I hope you followed the concepts so far. Any question for the moment? Feel free to contact me. Now let us continue. 

Click here to open the project directory, but read this page till end before doing anything with it.

As I told, the implementation of a transaction is simple. But the million dollar question is "How to do it?" . You may answer me quickly - "That is simple. By using Winsock Control". You are right. But let me clarify my question. "How to use Windows Sockets From Active Server Pages?". There is it. ASP doesn't provide any direct way to handle sockets. Worse, you need sockets to implement ECommerce transactions, and you need to use them from ASP. And that is what we are going to do now. Yes, we are going to develop a Socket wrapper component in Visual Basic, for using it in your ASP pages. Let me tell you one thing more. You can use this component not just for implementing ECommerce transaction - but for any kind of socket based communication. Of course, I am not providing all the concepts exploding in my mind :-). 
2. THE ARCHITECTURE
We will develop the component as a COM DLL. As you know, later, we can create instances of this socket component from ASP pages, to send and receive data. The project package with this article contains:
  1. SockObject - A well explained COM DLL, which shows you how to create a Wrapper class for using Sockets.
  2. Test Server - A standard EXE server for testing the functionality of SockObject, and for giving you the concept of writing a server. Very simple and well commented. It just echoes the data comes to it.
  3. An ASP file - A file named default.asp, for calling the SockObject to communicate with the server.

Please note these projects are interdependent. In our ASP file, we will:

  • Create an instance of the SockObject in the ASP file.
  • Sends the data to the Test Server, to get the reply.
  • Destroy the SockObject.
3. CONFIGURING THE PROJECT
To make the project up and running:
  1. Copy the default.asp file in the ASP Page directory, to any comfortable directory, and then create an Alias to it using Personal Web Server or Internet Information Server.
  2. Open and run the SockObject project in Visual Basic
  3. Open and run the Test Server project in Visual Basic
  4. Type the URL to the default.asp file
